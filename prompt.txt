<rol_y_objetivo>
Actúa como un Diseñador UI/UX Principal y Desarrollador Frontend Senior. Tu tarea es generar de forma completamente autónoma una Landing Page de un solo archivo (Single Page Application) lista para producción.
</rol_y_objetivo>

<contexto_del_proyecto>
- Nombre de la marca: "Interstellar Devs"
- Nicho: Agencia de desarrollo web y software de alto rendimiento con una identidad visual inspirada en el espacio y el rock.
- Tono del texto: Profesional, disruptivo, directo y redactado en español de Argentina (usando "vos").
- Estética Visual: Modo oscuro profundo (Dark Mode). Fondo principal #0B0C10, textos #C5C6C7, y acentos en cian neón (#66FCF1) para botones CTA y efectos de brillo (glow). Tipografías modernas de Google Fonts (ej. 'Space Grotesk' para títulos, 'Inter' para cuerpo).
</contexto_del_proyecto>

<estructura_requerida>
Genera la estructura en este orden exacto:
1. HEADER: Navegación flotante (sticky) con efecto blur de fondo. Logo a la izquierda, enlaces a la derecha (Servicios, Stack, Tripulación, Contacto).
2. HERO SECTION: Impactante. Título principal que utilice un efecto de máquina de escribir (typewriter) implementado en Vanilla JS. Subtítulo desafiante y un botón CTA cian neón.
3. DESCRIPCIÓN: Breve sección sobre nuestra filosofía de código limpio y despliegues a la velocidad de la luz.
4. SERVICIOS (Grid): 3 tarjetas con efecto hover (que se eleven y brillen los bordes) detallando: Desarrollo Frontend, Arquitectura Backend, y Despliegue en la Nube.
5. TESTIMONIALS: 2 o 3 reseñas de clientes simuladas en tarjetas minimalistas.
6. CONTACTO: Formulario maquetado (Nombre, Email, Frecuencia de transmisión, Botón de envío) centrado en una tarjeta con bordes sutilmente iluminados.
7. FOOTER: Derechos de autor, enlaces a redes sociales y un mensaje de cierre rockero.
</estructura_requerida>

<restricciones_tecnicas>
- HTML5, CSS3 y Vanilla JavaScript ÚNICAMENTE. No utilices frameworks (ni Tailwind, ni Bootstrap) ni librerías externas para JS.
- Todo el código CSS y JS debe estar incrustado (embedded) dentro del mismo archivo HTML utilizando las etiquetas <style> y <script>.
- Es obligatorio el uso de Flexbox o CSS Grid para la estructura responsiva (Mobile-First).
- Utiliza imágenes de marcador de posición (placeholders) de alta calidad insertando imágenes estables desde URLs confiables (ej. https://picsum.photos/800/600?random=1) o generadas por CSS con gradientes espaciales.
- EL FORMATO DE SALIDA DEBE SER EXCLUSIVAMENTE CÓDIGO. No envuelvas el resultado en bloques de markdown (html), no incluyas saludos, ni explicaciones previas o posteriores. Comienza con <!DOCTYPE html> y termina con </html>.
</restricciones_tecnicas>
